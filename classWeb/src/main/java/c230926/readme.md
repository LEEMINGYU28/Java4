#DAO, VO, DTO

## VO
- Value Object
- 값에 대한 객체
- 값이라는 건 변화하는걸까?
  -변하는 것에 이름 = 변수
  - 값을 변하지않는다. 변수가 바뀌는거지 값이 바뀌는건 아니다.
-getter만 존재

## DAO
- Data Access Object
- 데이터 접속 객체, DB에 요청을 보내기 위해서 사용하는 객체

## DTO
- Data Transfer Object 
- 계층(라우터)간에 데이터 전송을 위해 사용하는 객체

# Connection(커넥션) Pool 
- 생성해둔 객체를 가져다 쓰고 다 쓰면 반환한다.
- 사용할 때 마다 객체를 생성, 삭제하는 것이 아닌 풀에서 가져다 씀으로써
메모리 할당에 대한 시간을 절약할 수 있다.
